Lecture 6: Forms and Multimedia

Form Elements and Inputs

  • <form>: Defines the form and its attributes like action, method, and target
  • <input>: Text, password, radio, checkbox, submit, reset, button, color
  • <label>: Binds readable text to input elements
  • <select> and <option>: Drop-down menus
  • <textarea>: Multi-line input
  • <fieldset> and <legend>: Group related inputs

Form Input Attributes

  • value: Default input value
  • readonly: Prevents editing
  • disabled: Grays out and disables the field
  • size and maxlength: Control input width and character limit
  • HTML5 attributes: placeholder, required, pattern, autocomplete, autofocus, step, etc.

GET vs POST

  • GET: Sends form data in URL
  • POST: Sends data in request body, more secure for sensitive data
  • Use GET for short forms and POST for long or sensitive forms

Creating Effective Forms

  • Group fields logically using <fieldset>
  • Use label and placeholder for guidance
  • Design forms to be easy to use and accessible
  • Test across different devices and browsers

Audio in HTML

  • Use <audio> with controls to add playback options
  • <source> lets browsers choose compatible formats
  • Supported formats: MP3, Ogg, AAC

Video in HTML

  • Use <video> with controls, width, and height
  • Include fallback message for unsupported browsers
  • Supported formats: MP4, WebM, Ogg
  • autoplay, loop, and poster attributes available

Multimedia Tags Summary

  • <audio>: Embed sound with controls
  • <video>: Embed videos with playback options
  • <object> and <embed>: Include external multimedia like PDFs or Flash
